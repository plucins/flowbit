# Adapter: skill-definition

## Focus questions

- Are trigger conditions precise enough to avoid accidental misuse?
- Is the workflow deterministic where it must be?
- Are output contracts explicit and testable?
- Is extension guidance present for future contributors?

## Common failure patterns

- Ambiguous "when to use" rules.
- Overly long monolithic instructions with mixed concerns.
- Missing guardrails for risky operations.
- No progressive disclosure/reference split.

## Extra acceptance conditions

- Clear trigger scope and non-goals.
- Explicit output format contract.
- Extension steps documented for adding new cases.
